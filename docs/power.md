# MIXTEE: Power System

*← Back to [README](../README.md) | See also: [Hardware](hardware.md) · [System Topology](system-topology.md)*

------

> [!NOTE]
> **Why 5V?** Every load in the system runs at 5V or below (Teensy, NeoPixels, USB host, display, codecs via isolated DC-DC). Negotiating a higher USB PD voltage (9V, 12V, 20V) would require a buck converter — adding cost, board space, switching noise, and heat for zero gain. The 5V @ 5A USB PD supply delivers 25W, roughly 50% more than worst-case demand (~2.91A = 14.6W). There is no component that benefits from a higher rail voltage.

------

## Power Input

An **off-the-shelf STUSB4500 USB PD breakout module** provides power input. No custom power PCB.

### Module Options

- SparkFun Power Delivery Board (USB-C, Qwiic)
- STEVAL-ISC005V1 (ST evaluation board)
- Generic STUSB4500 breakout (AliExpress / Tindie)

### Configuration

- STUSB4500 NVM **pre-configured for 5V PDO only** (no higher voltages) via I2C before installation
- Fallback: 5.1kΩ CC resistors on module default to 5V/3A for non-PD supplies
- Module includes on-board decoupling and protection

### Mounting

- USB-C receptacle protrudes through back panel cutout (right side), labeled "PWR"
- Secured by USB-C jack nut or adhesive standoff
- Power only — D+/D- not routed

### Cable to Main Board

2-pin JST-PH or screw terminal, ~60–80 mm, 22 AWG minimum. Carries full system current (up to 5A).

| Wire | Signal | Notes |
|------|--------|-------|
| + | 5V (VBUS) | From STUSB4500 output, post-module protection |
| − | GND | Power return |

## Power Path

```
USB-C (PD supply)
  → STUSB4500 module (5V @ 5A negotiated)
    → 2-pin cable (JST-PH, 60-80 mm)
      → Polyfuse (2.5A hold / 5A trip, Main Board)
        → TPS22965 load switch (soft-start, 5A continuous)
          → 5V system rail
            ├── 5V_DIG  (USB hub, NeoPixels, display, all digital loads)
            └── 5V_ISO  (via 2× MEJ2S0505SC isolated DC-DC → analog domain)
                  └── ADP7118 LDO → 3.3V_A (ultra-low-noise codec/analog rail)
```

## Rail Distribution

### 5V_DIG (digital domain)

USB hub VBUS, NeoPixels, TFT backlight, and all digital loads. Noisy — kept physically and electrically separate from analog.

### 5V_ISO (isolated analog domain)

Generated by **2× Murata MEJ2S0505SC** isolated DC-DC converters (one per FFC, on the Main Board). 5V → 5V_ISO, 2W (400 mA) capacity each, 5.2 kV galvanic isolation. No copper connection to 5V_DIG or system GND.

Each Input Mother Board receives 5V_ISO via FFC and passes it through:
1. **Ferrite bead** (600Ω @ 100 MHz) + **10 µF ceramic** post-filter — attenuates DC-DC switching ripple
2. **ADP7118 LDO** — 5V_ISO → 3.3V_A, <10 µV RMS ultra-low-noise, powers codec AVDD/TVDD

Combined PSRR > 100 dB at the DC-DC switching frequency.

## Protection

| Protection | Location | Detail |
|------------|----------|--------|
| **Polyfuse** (2.5A hold / 5A trip) | Main Board, 5V rail entry | Overcurrent protection; power module may include its own |
| **TPS22965 load switch** | Main Board | 5A continuous, controlled slew rate limits inrush from bulk cap charging |
| **TPS2051 power switch** (×2) | IO Board | Per-port USB host VBUS current limiting (500 mA each) |
| **Bulk capacitor** (1000–2200 µF) | Near NeoPixel power entry | Buffers current spikes from up to 960 mA worst-case draw |
| **10 mΩ shunt resistor** | Main Board | Current measurement test point on main 5V rail |

## Soft-Latch Power Circuit

A **74LVC1G00 NAND gate** (SOT-23-5) wired as an SR latch provides clean power on/off without a mechanical power switch in the current path.

### Power On

Button press → sets latch → TPS22965 EN goes high → 5V rail on → Teensy boots → Teensy asserts KEEP_ALIVE (pin 41) to hold latch set.

### Power Off (clean)

Teensy detects button press (pin 40) → saves state → releases KEEP_ALIVE → latch resets → TPS22965 off.

### Power Off (hard)

Long-press > 4s → RC timeout resets latch directly (hardware failsafe, no firmware involvement).

### Sequencing Diagram

```
Button press → SR latch SET → TPS22965 EN → 5V rail on → Teensy boots
                                                           ↓
                                           Teensy asserts KEEP_ALIVE (pin 41)
                                           Teensy reads button via pin 40

Button press → Teensy detects → saves state → releases KEEP_ALIVE
                                                           ↓
                                           SR latch RESET → TPS22965 off
Long-press (>4s) → RC timeout resets latch directly (hardware failsafe)
```

### Components

- 74LVC1G00 NAND gate (SOT-23-5) — SR latch
- Debounce capacitor on button input
- RC timeout network for hard power-off
- Pull-up/pull-down resistors for latch stability

## Power Button

Off-the-shelf **momentary push button** with screw-collar panel mount. Mounted on the back panel next to the PWR USB-C (right side). Labeled "POWER".

| Wire | Signal | Teensy Pin | Notes |
|------|--------|-----------|-------|
| 1 | Button sense | 40 | Teensy reads press events |
| 2 | KEEP_ALIVE | 41 | Teensy holds latch set during normal operation |
